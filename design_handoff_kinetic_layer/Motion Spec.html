PWRL Interactive Layer

Motion Spec v1

The contract for implementation. Reference build: motion/motion.css + motion/pwrl-motion.js (drop-in, dependency-free). Working demos: Home.html, Fund.html, Cursor Recipes.html. Status: committed 06.11.26 — shipped configuration: Standard intensity, Recipe A (Lumen), glow #085CF0, magnetic pull + sheen, per-line hero rise, Ken Burns + parallax on.

1 · Principles

Precision — deliberate easings, orchestrated staggers, data that behaves like data. Economy — fast in, faster out; the site feels quick because the motion is quick. Weight — short distances, confident ease-out, no bounce, no elastic, no rotation. One vocabulary across all six pages. Play lives exclusively in the cursor layer; everything is serious at rest.

Hard rule: the motion layer is purely additive — transform, opacity, masks, and pseudo-element glows. Nothing that changes box geometry. A screenshot of any resting state is pixel-identical to the static build. Without JS, nothing is ever hidden (all pre-states are gated on html[data-mo-on], which only the script sets).

3 · Reveal vocabulary

Markup contract: add data-mo (rise), data-mo="fade", data-mo="draw" (scaleY) or data-mo="draw-x" (scaleX). A parent data-mo-stagger auto-indexes children; --mo-i sets explicit order, --mo-d a fixed head-delay. Reveals fire once, on first view (IntersectionObserver, rootMargin: 0 0 -8%).

  • Rises snap: 24px travel + 0.97→1 scale at expo-out — high initial velocity reads as punch; still no bounce, no overshoot.
  • Section headings lead; body copy and CTAs follow at the stagger interval.
  • Grids cascade left-to-right / top-to-bottom via data-mo-stagger.
  • Hero zones are wrapped in data-mo-hero: they enter on load, never wait for scroll.

4 · Cursor layer

All cards share: lift translateY(-3px) at 140ms + a pointer-driven pseudo-element (--mx/--my px, --dx/--dy −1..1, set by one rAF-throttled pointermove listener). Shipped recipe: A · Lumen (html[data-glow="lumen"], the default the script sets). Cast and Edgelight remain implemented and selectable for future revisits; the comparison lives in Cursor Recipes.html.

C · Edgelight (archived)Masked 1.5px rim brightens nearest the pointer + faint washInstrumentation, most restrained

Buttons

  • Magnetic micro-pull: ≤6px × strength toward the cursor, JS transform, 140ms reset. Shipped strength: 0.3 (≈1.8px max pull).
  • Sheen sweep: clipped white strip (32% alpha) sweeps once per hover, 450ms.
  • Arrow nudge: .mo-arrow spans translate 4px on hover of the parent link/card.

6 · Reduced motion & mobile

prefers-reduced-motion

  • Every reveal, glow, sweep, count-up, parallax and Ken Burns is disabled; content is fully visible immediately.
  • The pre-existing loops (hero rotator, quote crossfade, sector wheel) stay stilled exactly as the current globals.css guards do — the layer extends that pattern.
  • QA simulator: Tweaks → “prefers-reduced-motion” forces html[data-mo-reduced], which mirrors the media query rules 1:1.

Touch / mobile

  • Scroll reveals stay, with travel shortened to 12px.
  • Cursor glows and magnetism are inert (hover: none); cards acknowledge taps with lift + glow on :active.
  • QA simulator: Tweaks → “Touch device”.

7 · Performance constraints

  • Animate only transform and opacity; glows are pseudo-elements faded by opacity — box-shadow is never animated.
  • One document-level pointermove (rAF-throttled), one IntersectionObserver, passive scroll listeners. No scroll-jacking, no wheel hijack, no canvas/WebGL.
  • Reveals fire once and unobserve; will-change only on persistently animated elements.

8 · Repo integration (Next.js App Router + Tailwind v4)

  • Copy motion/motion.css → frontend/src/app/motion.css and motion/motion-bespoke.css → frontend/src/app/motion-bespoke.css; @import both from globals.css (after the theme block, global first).
  • Copy motion/pwrl-motion.js → frontend/public/pwrl-motion.js and load it once in app/layout.tsx via <Script src="/pwrl-motion.js" strategy="afterInteractive" />. No component needs to become a client component — the contract is data attributes and classes, all serializable from server components.
  • Annotate server components with data-mo / data-mo-stagger / .mo-card / .mo-btn exactly as in the two prototype pages (they mirror the repo's block structure 1:1: Hero, Intro, PullQuote, Timeline, PortfolioBlock, StockInfo, Philosophy, FAQBlock, …).
  • SectorWheel: add .mo-wheel / .wheel-orbit / .wheel-icon-inner class hooks (markup only; the stock CSS animation remains the no-JS/reduced fallback).
Out of scope by design: mobile menu choreography (existing behavior kept), filings-table interactions (legal), and any treatment that alters layout, scroll position, or the 27s rotator timing.
